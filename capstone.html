CAST Medical Interface

A patient simulation system for nursing education that transforms a standard CPR manikin into an interactive diagnostic training tool. I built a custom RFID-enabled stethoscope that plays realistic heart and lung sounds based on instructor-configured conditions.

The Problem

Nursing students need to learn auscultation — listening to heart and lung sounds to diagnose conditions. Traditional training relies on expensive high-fidelity manikins or real patients, both of which have limitations:

  • High-fidelity manikins cost tens of thousands of dollars
  • Students can't practice the same condition repeatedly
  • Instructors have limited control over the learning scenario
  • Real patients present ethical and availability constraints
Challenge

Create an affordable, controllable patient simulator that gives nursing students realistic practice identifying heart and lung conditions — without the cost of commercial solutions.

The Solution

I designed a modular system with three components: a custom RFID-enabled stethoscope, a Raspberry Pi that processes RFID reads and plays audio, and a web interface for instructors and students.

System Architecture

The system uses a modular design where different "modules" can be swapped into a base manikin unit. Each module contains RFID tags at specific auscultation points. When the custom stethoscope (with embedded RFID reader) is placed on these points, it triggers the appropriate sound.

System architecture: Modular design allows different training scenarios

The architecture separates concerns cleanly:

  • Hardware Layer: RFID reader in stethoscope housing, connected to Raspberry Pi
  • Processing Layer: Python script on Pi reads RFID tags, queries current condition from server, plays appropriate audio
  • Interface Layer: Flask web app with separate dashboards for instructors and students

Custom Stethoscope

I modified a standard stethoscope housing to contain an RC522 RFID reader module. The reader connects to the Raspberry Pi and continuously scans for tags. When a tag is detected, it sends the location data to the main script.

Custom stethoscope housing with RC522 RFID reader and earphone output

The stethoscope includes earphone output so students hear sounds directly through the device — mimicking real clinical use. This creates a more immersive training experience than playing sounds through external speakers.

User Interfaces

The web application provides two distinct interfaces tailored to different user needs:

Instructor Dashboard: Control active conditions
Student Dashboard: Real-time simulation status

Instructor Dashboard

Instructors can set the current condition (from 13 heart and 15 lung options), activate/deactivate the auscultation module, and monitor student interactions. The interface shows real-time status of what sound is playing and which location the stethoscope is at.

Student Dashboard

Students see a simplified view focused on the learning experience. They can see when the simulation is active and ready for examination, access quiz modes, and review session history for self-assessment.

Technical Details

The backend runs on Flask with endpoints for condition management and status updates. The Raspberry Pi script polls these endpoints and manages audio playback through pygame.

Key Technical Decisions
  • RFID over alternatives: Chose RFID tags for durability and no line-of-sight requirement — students can "auscultate" through the manikin skin naturally
  • Flask for simplicity: Lightweight server that runs on the Pi itself, no external hosting needed
  • Modular audio library: Sounds organized by condition type, easily expandable for new conditions
  • Real-time polling: Pi checks condition state every 100ms for responsive feedback

Impact

Results

The system provides nursing students with repeatable, controllable practice scenarios at a fraction of the cost of commercial solutions. Instructors can now create specific learning scenarios and students can practice identifying conditions until they're confident.

This project demonstrated the value of understanding the end user's workflow. By working directly with nursing faculty, I was able to design an interface that fits naturally into their teaching methods rather than forcing them to adapt to the technology.

What I Learned

  • Hardware-software integration: Debugging across the full stack from RFID reads to web interface required systematic troubleshooting
  • User-centered design: Early conversations with nursing faculty shaped every design decision
  • Modular architecture pays off: The ability to add new conditions or swap modules made iteration fast
  • Real-time systems: Managing latency between RFID read, condition lookup, and audio playback was critical for immersion
