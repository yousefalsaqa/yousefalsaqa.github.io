Energy Digital Twin Application

A full-stack digital twin platform that combines EnergyPlus building simulation with custom predictive analytics. The system lets engineers model building energy consumption, run what-if scenarios, and forecast future usage patterns through an interactive dashboard.

Type
Research Project
Domain
Industrial Energy
Stack
Python FastAPI EnergyPlus Streamlit

The Problem

Building energy management is complex. Facility managers need to understand how changes to HVAC settings, occupancy patterns, or equipment schedules affect energy consumption — but testing these changes in real buildings is expensive and disruptive.

Traditional energy modeling tools like EnergyPlus are powerful but require expertise to use. There's a gap between the detailed simulation capabilities and what non-experts can access for day-to-day decision making.

The Solution

I built a digital twin application that wraps EnergyPlus simulation with a modern web interface and adds predictive capabilities on top. The system has three main components:

  • Simulation Engine: Generates energy consumption data using configurable baseline and volatility parameters, or runs full EnergyPlus building simulations
  • Prediction Module: Trains machine learning models on simulation data to forecast future consumption patterns
  • Interactive Dashboard: Streamlit frontend that lets users configure simulations, visualize results, and explore predictions

Architecture

System Design

The backend exposes RESTful API endpoints via FastAPI. The simulation module can run in two modes: a lightweight baseline-volatility model for quick what-if analysis, or a full EnergyPlus simulation for detailed building modeling. The Eppy library handles IDF file manipulation for EnergyPlus integration.

Technical Details

Simulation Module

The simulation engine generates time-series energy data using stochastic modeling. Users specify baseline consumption, volatility (how much the consumption varies), and simulation duration. The output feeds into both visualization and the prediction pipeline.

EnergyPlus Integration

For detailed building simulation, the system modifies IDF (Input Data File) templates to adjust parameters like thermostat setpoints, then runs EnergyPlus as a subprocess. Output CSV files are parsed back into the application for analysis. This allows modeling of specific buildings with real weather data.

Predictive Analytics

The prediction module trains a linear regression model on simulation outputs. While simple, this provides a baseline for forecasting. The architecture is designed to be extensible — swapping in more sophisticated models (LSTM, Prophet) requires minimal changes.

Results

The platform provides a unified interface for energy analysis that previously required juggling multiple tools. Engineers can:

  • Run quick what-if scenarios in seconds using the baseline model
  • Execute detailed EnergyPlus simulations for specific buildings
  • Train predictive models on historical or simulated data
  • Visualize consumption patterns and predictions interactively
