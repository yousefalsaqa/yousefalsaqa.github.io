Terrabot CO2 Monitor

An autonomous indoor robot that maps room CO2 levels to identify ventilation dead zones. Using SLAM for navigation and a precision air quality sensor, Terrabot generates heat maps showing CO2 distribution — helping facility managers optimize HVAC systems and improve air quality.

Course
MREN 203
Team
Terraforma Corp.
Stack
ROS Python SLAM Raspberry Pi

The Problem

Poor indoor air quality is a hidden problem. High CO2 levels cause drowsiness, reduced cognitive function, and discomfort — but building occupants usually can't perceive it until levels are dangerously high. Traditional air quality monitoring uses fixed sensors that miss localized problems.

We asked: what if a robot could autonomously survey an entire room and create a map of where CO2 accumulates? This would reveal ventilation dead zones, overcrowded areas, and HVAC inefficiencies that fixed sensors miss.

The Robot

Terrabot with LiDAR, CO2 sensor, and Raspberry Pi mounted

Terrabot is built on a differential-drive platform with two powered wheels and a castor. The chassis is custom-designed to mount all sensors at optimal heights — the LiDAR at chest height for obstacle detection, and the CO2 sensor at breathing height where air quality matters most.

SLAM Navigation

The robot uses Hector SLAM to simultaneously map the room and track its position. A 360-degree LiDAR sensor provides the raw distance measurements, which the SLAM algorithm processes to build an occupancy grid map in real-time.

Real-time map generated by Hector SLAM from LiDAR data

Navigation uses Dijkstra's algorithm to plan paths through the mapped space. The robot can operate in two modes: autonomous exploration (covering the entire accessible area) or remote control for targeted measurements in specific locations.

CO2 Mapping

As Terrabot navigates, it continuously logs CO2 readings with their corresponding map coordinates. After surveying a room, this data is processed to generate a heat map visualization:

Heat map showing CO2 distribution — red indicates high concentration areas

The visualization uses interpolation to fill gaps between measurement points, creating a continuous surface that facility managers can immediately understand. High-CO2 zones appear in red, indicating areas that need better ventilation or reduced occupancy.

System Architecture

ROS-based architecture with separate nodes for SLAM, sensing, and control

The software runs on ROS (Robot Operating System), with separate nodes handling:

  • SLAM Node: Processes LiDAR data and maintains the map
  • CO2 Node: Reads from the SCD30 sensor and publishes timestamped measurements
  • Navigation Node: Plans paths and sends motor commands
  • Control Node: Handles mode switching and user interface

Hardware

Components
  • Raspberry Pi 4: Main compute unit running ROS
  • Arduino Uno WiFi Rev2: Low-level motor control and sensor interface
  • RPLiDAR A1: 360-degree scanning at 8000 samples/second
  • Sensirion SCD30: CO2, temperature, and humidity sensor with ±30ppm accuracy
  • Sharp IR Sensors: Close-range obstacle detection for collision avoidance

The dual-processor architecture splits responsibilities: the Pi handles computation-heavy tasks (SLAM, path planning, data logging) while the Arduino manages real-time motor control and sensor polling. Communication between them uses serial over USB.

Results

Terrabot successfully demonstrated autonomous room surveying and CO2 mapping. In testing, it identified ventilation dead zones that weren't apparent from building plans — areas where CO2 accumulated 40% higher than the room average.

The project showed me how to integrate multiple complex systems (SLAM, sensor fusion, path planning) into a working robot. The ROS architecture skills transferred directly to my later work on autonomous vehicles.
