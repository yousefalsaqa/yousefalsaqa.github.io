Terrabot Project

By Terraforma Corporation for Q-PMC

Executive Summary

The Terrabot project involves the design and development of a robot for CO2 monitoring in indoor environments. Using SLAM and CO2 sensors, it generates a “heat map” of CO2 distribution, enhancing air quality management. This solution is expected to reduce costs associated with air quality monitoring while ensuring a healthier indoor environment.

System Architecture

The Terrabot uses a Raspberry Pi for data processing and connects to various sensors, including LiDAR for SLAM and a CO2 sensor. Key components communicate through ROS nodes. Navigation modes (autonomous or remote control) are selectable, and the robot utilizes Dijkstra's Algorithm for effective mapping.

CO2 Mapping

The CO2 mapping feature employs a heatmap visualization technique, using measured CO2 concentration data at specific coordinates. Data interpolation fills gaps between measurements, creating a comprehensive visual representation of CO2 distribution in the environment.

Room Mapping

The room mapping feature is powered by Hector SLAM, which uses LiDAR data to build a real-time 2D occupancy grid map. This allows the robot to autonomously navigate the room while avoiding obstacles, ensuring efficient and accurate mapping for CO2 heatmap generation.

Hardware

  • Raspberry Pi v4
  • Arduino Uno Wifi Rev2
  • 360-Degree LiDAR Sensor
  • SCD30 Air Quality Sensor
  • SHARP Distance Sensor
