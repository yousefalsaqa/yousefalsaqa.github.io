Formula SAE Steering System

Designed and built the steering wheel electronics for Queen's Formula SAE racing team. The system features a custom PCB, integrated TFT display showing real-time telemetry, and CAN Bus communication with the vehicle's ECU and data acquisition system.

Team
Queen's Formula SAE
Role
Electronics Lead
Stack
C++ CAN Bus PCB Design Embedded

The Challenge

Formula SAE is a collegiate engineering competition where students design and build formula-style race cars. The steering wheel is the primary interface between driver and machine — it needs to display critical information (speed, gear, temperatures, warnings) while being compact, reliable, and easy to read at high speed.

The previous steering wheel system was outdated and lacked integration with the car's data acquisition system. The team needed a new solution that could display real-time telemetry and communicate reliably with other vehicle systems.

My Role

I was responsible for the complete electronics package:

  • PCB Design: Designed and routed a custom 4-layer PCB to fit within the steering wheel housing, integrating microcontroller, display driver, CAN transceiver, and power regulation
  • Display Interface: Programmed graphics using the FT800/FT813 GPU to render gauges, shift lights, and warning indicators at 60fps
  • CAN Integration: Implemented the CAN Bus protocol stack to receive data from the ECU and transmit button states to the data acquisition system
  • Firmware: Wrote all embedded code in C++, including display rendering, CAN message parsing, and state machine logic for different display modes

Display System

Steering wheel with integrated TFT display showing real-time telemetry

The TFT display shows the driver everything they need at a glance:

  • Center: Current gear position (large, easy to read)
  • Top bar: RPM indicator with color-coded shift lights
  • Left gauge: Coolant temperature with warning threshold
  • Right gauge: Oil temperature and pressure
  • Bottom: Speed, lap time, and system status

Graphics rendering uses the FT813 GPU's hardware acceleration for smooth animations even during rapid data updates. The shift lights transition from green through yellow to red as RPM approaches the shift point, giving the driver intuitive feedback.

CAN Bus Communication

Protocol Implementation

The steering wheel communicates on the vehicle's CAN network at 500kbps. It receives telemetry broadcasts from the ECU (RPM, temperatures, gear position) and transmits button states when the driver activates controls like pit limiter or traction control adjustment.

Message filtering ensures the microcontroller only processes relevant CAN IDs, reducing CPU load. Error handling includes automatic bus-off recovery and watchdog monitoring to maintain reliability during races.

Hardware Design

The custom PCB packs significant functionality into a compact form factor:

  • Microcontroller: STM32F4 running at 168MHz with hardware CAN peripheral
  • Display Driver: FTDI FT813 graphics controller with SPI interface
  • CAN Transceiver: MCP2551 with ESD protection for automotive environment
  • Power: Switched-mode regulator accepts 12V vehicle power, outputs 3.3V and 5V rails
  • Buttons: Debounced inputs for paddle shifters and auxiliary controls

The PCB design went through three revisions before finalizing. Key lessons included proper ground plane separation between digital and analog sections, and adequate decoupling on the power rails to prevent noise from affecting display quality.

Results

Competition Performance

The steering system performed reliably throughout the competition season with zero electronic failures. Drivers reported improved awareness of vehicle state, particularly appreciating the intuitive shift light progression and clear warning indicators.

This project taught me the full embedded systems development cycle — from schematic capture and PCB layout through firmware development and on-vehicle testing. The skills transferred directly to my later work on the AutoDrive autonomous vehicle team.
