Universal Admission Analysis Dashboard

A comprehensive analytics platform that processes 10 years of admission data to uncover patterns in student performance. The insights from this analysis directly contributed to policy changes in Commerce admission requirements.

Developer & Analyst

The Problem

University admissions teams make decisions that affect thousands of students yearly, but often lack the analytical tools to understand long-term trends. Questions like "Do high school grades actually predict university success?" require deep data analysis that wasn't easily accessible.

  • Admission thresholds were based on historical precedent, not data
  • No easy way to compare high school performance vs. university outcomes
  • Policy decisions lacked quantitative backing
  • Data existed but was siloed in Excel files across departments
Challenge

Build a tool that could analyze a decade of admission data, reveal patterns between high school predictors and university success, and enable "what-if" policy modeling — all while handling sensitive student data appropriately.

The Solution

What started as a simple Python script evolved into a full desktop application with four distinct analysis modules. I designed it to be config-driven so the same tool could be used across different programs (Commerce, Engineering, Arts, etc.) without code changes.

Main Dashboard: Program selection, data overview, and analysis controls

Dashboard Modules

1. Main Analysis Dashboard

The default view loads program data and generates visualizations comparing high school metrics (Math, English, overall average) against first-year university GPA. Statistical rigor is built in — correlations are calculated with bootstrap validation to ensure findings are significant.

2. Course-Level Analysis

Goes deeper than program-level by analyzing individual university courses. Instead of "how do Commerce students do overall?", this answers "how do students perform in COMM 101 based on their high school Math grade?" This granularity revealed that some courses are better predicted by specific high school subjects.

Course Analysis: Per-course GPA vs. high school performance correlations

3. Policy Threshold Calculator

This is where the tool became powerful for decision-making. Users can adjust admission thresholds (Math minimum, English minimum, overall average) and immediately see how many students from the historical dataset would have been admitted or rejected under the new policy — with color coding showing the impact.

How It Works

The calculator takes the full 10-year dataset and applies hypothetical admission rules. For each threshold combination, it shows: students who would still be admitted, students who would be rejected, and predicted first-year performance of each group based on historical correlations.

4. Student Lookup

All analysis was done on anonymized data, but I built a reverse-engineering function for the anonymization scheme so individual students could be looked up when needed. This allowed admissions staff to drill down from aggregate trends to specific cases.

Technical Approach

Config-Driven Design

Rather than hardcoding program-specific logic, all program definitions live in configuration files. Adding a new program to analyze is as simple as creating a new config specifying the data file path, relevant columns, and any program-specific thresholds. This made the tool truly "universal."

The technical stack was chosen for rapid development and ease of deployment on university machines:

  • Tkinter for GUI: Native Python, no installation required on target machines
  • Pandas for data: Handles the 35k-row datasets efficiently with familiar DataFrame operations
  • Matplotlib for visualization: Publication-quality charts that could be exported for presentations
  • Statistical validation: Bootstrap resampling to ensure correlations aren't artifacts of the data

Impact

Policy Change

Analysis from this dashboard directly contributed to changes in Commerce admission requirements. Minimum thresholds for Math and English were raised from 80 to 85, and the overall average minimum increased from 87 to 90.

The data showed that students admitted below these thresholds had significantly lower first-year GPAs and higher rates of academic difficulty. The policy change wasn't arbitrary — it was backed by a decade of evidence.

Beyond the policy change, the dashboard continues to be used for:

  • Recruitment strategy — understanding which regions and schools produce successful students
  • Program planning — identifying courses where students consistently struggle
  • Ad-hoc analysis — answering specific questions from faculty and administration

What I Learned

  • Data quality matters: Spent significant time cleaning and normalizing 10 years of inconsistently formatted data
  • Build for non-technical users: The people using this tool aren't programmers — the interface had to be intuitive
  • Config over code: Making the tool program-agnostic meant it could be reused immediately
  • Sensitivity handling: Working with student data requires careful attention to privacy and access controls
  • Impact requires communication: The analysis was only valuable because it was presented in a way decision-makers could understand
